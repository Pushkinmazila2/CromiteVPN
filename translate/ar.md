# Stealth Browser (قائم على Cromite)

متصفح أندرويد خاص يعتمد على **Cromite** مع سلسلة بروكسي محلية مدمجة عبر **Xray-core (VLESS + Reality)**.

تم تصميم هذا المشروع لتوجيه جميع حركة مرور المتصفح عبر طبقة بروكسي داخلية **دون الحاجة إلى استخدام خدمة Android VpnService**.

---

## 🚀 الميزات الرئيسية

*   **تكامل Xray الأصلي:** تم دمج نواة Xray مباشرة في شبكة Chromium عبر المقابس المحلية (local sockets). يتم تغليف كل حركة المرور داخل عملية واحدة.
*   **لا يتطلب VpnService:** لا ينشئ المتصفح واجهة VPN على مستوى النظام. لا توجد أيقونة "المفتاح" في شريط الحالة، ولا يمكن للنظام أو التطبيقات الأخرى اكتشاف استخدام البروكسي.
*   **تخفي VLESS + REALITY:** يدعم بروتوكول REALITY بشكل أساسي. بالنسبة للمراقبين الخارجيين (مزود الخدمة أو أنظمة DPI)، تبدو حركة المرور كجلسة TLS قياسية لمورد موثوق (مثل Microsoft أو Google).
*   **البطارية والأداء:** من خلال تجنب اعتراض الحزم على مستوى واجهة TUN، يتم تقليل استهلاك المعالج وتوفير طاقة البطارية بشكل ملحوظ.
*   **منع تسريب DNS:** يتم معالجة جميع استعلامات DNS بواسطة وحدة DNS المدمجة في Xray وإرسالها عبر النفق المشفر، مما يمنع التسريبات لدى مزود الخدمة.
*   **حركة مرور مخفية:** أدوات تحليل حركة المرور على الجهاز (مثل PCAPdroid) لن تظهر سوى اتصال واحد بعناوين IP الخاصة بخادمك مع SNI مموه.

## 🛠 التنفيذ التقني

في هذا الإصدار، تم تنفيذ تغليف كامل لحركة المرور:
1. تقوم طبقة الشبكة `Cronet` بتوجيه الطلبات إلى مثيل Xray الداخلي.
2. يتم استخدام **الوضع العام (Global Mode)**: تمر جميع حركة مرور المتصفح دون استثناء عبر مخرج VLESS المكون.
3. حماية معززة ضد تسريبات WebRTC عبر سياسة `ice_candidate_policy` صارمة.

## 📥 الإعدادات

في المرحلة الحالية، يتم التكوين من خلال واجهة إعدادات المتصفح:
1. انتقل إلى **Settings** -> **Privacy and Security**.
2. اختر **Xray Configuration**.
3. الصق تكوين JSON الخاص بك (يدعم VLESS + Reality).
4. أعد تشغيل التطبيق لتطبيق الإعدادات.
